# MEMORY.md

## Purpose
This file explains how memory works for this project.

## Operational memory
Daily or session-specific notes belong in:

- `memory/YYYY-MM-DD.md`

Use these notes for:
- what was done today
- blockers
- next steps
- transient discoveries
- partial decisions that may change

## Durable project memory
Stable project knowledge belongs in:

- `project-memory/`

Use this for:
- accepted terminology
- stable design decisions
- durable project facts
- settled structure choices

## Rules
- Do not leave important decisions only in chat.
- Do not put transient daily notes into durable memory.
- Do not treat this file as a dump of project facts.
