## Known problems observed

- Math context detection is not working — snippets flagged as math-only (e.g.
  `te` → `\exists`) are firing outside math blocks in plain text.
- The options mapping from shortcuts.json (`t`, `m`, `tA`, `mA`, `r`, `w`)
  was not correctly translated to LuaSnip conditions and snippet types.
- Auto-trigger is firing indiscriminately — the `condition = in_math` guard
  is not being respected for autosnippets.
- Many trigger conflicts were not resolved (e.g. duplicate `lim`, `or`, `and`,
  `NN`, `dne`, `qed`, `st` all appear multiple times in shortcuts.json with
  different modes, none handled correctly).
- The `from_lua` loader return format (`snips, autosnips`) and the interaction
  between `snippetType = "autosnippet"` set on items in the first table vs. the
  second table was not verified.

## What to do when redoing

1. Verify math context detection works in isolation before adding any snippets.
2. Port one category at a time, test each before moving on.
3. Carefully map each option flag:
   - `t` = text mode only (outside math)
   - `m` = math mode only (condition = in_math)
   - `A` = autosnippet (snippetType = "autosnippet")
   - `r` = regex trigger (regTrig = true, wordTrig = false)
   - `w` = word boundary (wordTrig = true, which is the default)
   - `tA` = global autosnippet with no math condition
   - `mA` = math autosnippet
4. Resolve all duplicate triggers explicitly (pick one or use priority).
5. Test the `from_lua` loader return format is correct.
