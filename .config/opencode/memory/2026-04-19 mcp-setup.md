## Session: Playwright with system browser profile

### What we did
- Investigated how Playwright MCP can use an existing browser profile
- Explored CDP + Firefox workflow but Firefox 129+ deprecated CDP support
- Explored using Chrome Extension (requires Chrome browser installed)
- Decided on: Playwright's bundled Chromium + user's Chromium profile

### Final configuration
- Playwright uses `--browser=chromium` (bundled Chromium, auto-installed)
- Uses existing profile at `~/.config/chromium/Default`
- No system Chrome/Chromium needed for user — only for Playwright
- User continues using Firefox as primary browser — no conflicts

### Profile reasoning
- User doesn't run Chromium as regular browser → no profile lock
- Playwright launches its own Chromium → accesses profile without conflicts
- This is ideal: "your Firefox" + "playwright's Chromium" separation
- Future: can swap in Chrome profile from another machine by updating `--user-data-dir`
